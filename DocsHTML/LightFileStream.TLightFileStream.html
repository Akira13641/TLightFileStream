LightFileStream

Declaration

type TLightFileStream = record

Description

The sole type implemented by this unit so far.

Returns a pointer to itself from all functions to avoid unnecessary allocation or copying and to allow for a convenient "method chaining" API.

Description

Nested Types

TFileState = (...);

Determines whether read or write operations are actively accepted.

All functions return immediately without doing anything if the current state is "wrong" relative to them, unless {$define NoChecks} is set, which improves performance but is somewhat less "safe."

Values
  • fsReading
  • fsWriting

Fields

FFileName: PChar;

The name of the underlying file on disk.

FHandle: THandle;

The handle of the underlying file on disk.

FOpen: Boolean;

Indicates whether the file is open and can be accessed at all.

All functions return immediately without doing anything in the event that this is false, unless {$define NoChecks} is set, which improves performance but is somewhat less "safe."

FState: TFileState;

A private instance of TFileState, used as described above.
